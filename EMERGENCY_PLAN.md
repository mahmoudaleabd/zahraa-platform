# خطة الطوارئ (Emergency Plan)

## 🚨 سيناريوهات الطوارئ

### 1. تعطل الموقع بالكامل

#### الأعراض:
- الموقع لا يستجيب
- صفحة خطأ 500
- الموقع لا يتحمل

#### الإجراءات الفورية:
1. **فحص حالة Vercel**:
   - اذهب إلى [Vercel Dashboard](https://vercel.com/dashboard)
   - تحقق من حالة النشر الأخير
   - راقب logs النشر

2. **فحص حالة Supabase**:
   - اذهب إلى Supabase Dashboard
   - تحقق من حالة قاعدة البيانات
   - راقب logs الاستعلامات

3. **إعادة النشر**:
   ```bash
   vercel --prod
   ```

4. **التواصل مع الدعم**:
   - واتساب: [رقم الدعم]
   - إيميل: [إيميل الدعم]

---

### 2. تسرب البيانات أو اختراق الأمني

#### الأعراض:
- نشاط غير عادي في قاعدة البيانات
- تقارير من المستخدمين عن مشاكل في الحسابات
- ظهور بيانات حساسة

#### الإجراءات الفورية:
1. **تغيير جميع مفاتيح API**:
   - غيّر `NEXT_PUBLIC_SUPABASE_ANON_KEY` في Vercel
   - غيّر `service_role_key` في Supabase
   - أعد نشر التطبيق

2. **فحص سياسات RLS**:
   - راجع جميع سياسات Row Level Security
   - تأكد من عدم وجود سياسات مفتوحة

3. **مراجعة Logs**:
   - راجع logs Supabase للنشاط المشبوه
   - راجع logs Vercel للطلبات غير المعتادة

4. **إبلاغ المستخدمين**:
   - أرسل إشعار للمستخدمين المتأثرين
   - اطلب منهم تغيير كلمات المرور

---

### 3. فقدان البيانات

#### الأعراض:
- بيانات مفقودة من قاعدة البيانات
- صور مفقودة من Storage
- جداول فارغة

#### الإجراءات الفورية:
1. **استعادة النسخة الاحتياطية**:
   - اذهب إلى Supabase Dashboard
   - Database > Backups
   - اختر آخر نسخة احتياطية سليمة
   - اضغط Restore

2. **فحص سبب الفقدان**:
   - راجع logs لمعرفة السبب
   - تحقق من سياسات RLS
   - راجع الكود الأخير المدمج

3. **منع التكرار**:
   - أضف المزيد من النسخ الاحتياطية
   - راجع صلاحيات المستخدمين
   - أضف logs إضافية

---

### 4. مشاكل في الأداء

#### الأعراض:
- الموقع بطيء جداً
- استهلاك عالي للموارد
- timeouts متكررة

#### الإجراءات الفورية:
1. **فحص الأداء**:
   - استخدم Vercel Analytics
   - راجع Supabase Performance
   - فحص استعلامات البطيئة

2. **تحسين الصور**:
   - تأكد من ضغط الصور
   - استخدم WebP format
   - قلل حجم الصور

3. **تحسين قاعدة البيانات**:
   - أضف فهارس (indexes) للجداول
   - حسّن الاستعلامات البطيئة
   - استخدم caching

4. **توسيع الموارد**:
   - ترقية خطة Supabase
   - تفعيل Edge Functions
   - استخدام CDN

---

### 5. مشاكل في المصادقة

#### الأعراض:
- المستخدمون لا يستطيعون تسجيل الدخول
- مشاكل في إنشاء الحسابات
- جلسات منتهية بشكل غير متوقع

#### الإجراءات الفورية:
1. **فحص Supabase Auth**:
   - راجع إعدادات Authentication
   - تحقق من Email templates
   - راجع سياسات الجلسات

2. **فحص متغيرات البيئة**:
   - تأكد من صحة `NEXT_PUBLIC_SUPABASE_URL`
   - تأكد من صحة `NEXT_PUBLIC_SUPABASE_ANON_KEY`

3. **إعادة نشر التطبيق**:
   ```bash
   vercel --prod
   ```
